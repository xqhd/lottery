# 年会抽奖系统 — 详细开发计划

## 1. 概要

本项目目标为开发一款“**高端大气上档次**”的年会抽奖软件/网页（以下简称“系统”），支持：自定义奖项与中奖概率、导入人员名单（Excel / CSV / TXT 等）、自定义图片 / 动图背景、循环播放自定义音乐、导出抽奖结果、全程可审计与回放等功能。产品既可用于公司年会现场展示（大屏/投影），也可作为线上抽奖工具。

------

## 2. 项目目标（可衡量）

- 功能完备：覆盖导入名单、奖项管理、加权抽奖、媒体自定义、结果导出、审计日志等核心功能。
- 品质与体验：动画与音效流畅，界面视觉高端（可配置皮肤）、适配大屏幕与移动端。
- 公平可信：抽奖算法公开、可复现（提供随机种子与抽奖日志）。
- 可部署性：支持单机部署（Docker）与云端部署（容器服务）。
- 可维护性：单元/端到端测试覆盖关键逻辑，代码结构清晰。

------

## 3. 核心功能清单

### 3.1 抽奖管理

- 新建/编辑抽奖活动（活动名称、时间、描述）。
- 奖项管理：添加奖项（名称、数量、等级、是否可重复中奖）、奖项图片/动画、奖项权重（概率）或直接指定中奖数量。
- 支持设置“同一人限中次数”或“不同奖项排除规则”。

### 3.2 名单导入与管理

- 支持导入格式：Excel (.xlsx/.xls)，CSV，纯文本（每行一名），以及手动粘贴名单。
- 导入字段支持：姓名、工号/ID、部门、联系方式、自定义字段。
- 导入过程提供预览/字段映射/去重规则（按姓名+工号/ID）。
- 支持按部门/标签筛选抽奖候选池。

### 3.3 媒体与舞台设置

- 背景：上传图片或动图（gif / mp4），支持裁剪/适配/保持纵横比、蒙版与淡入淡出过渡。
- 音乐：上传音频 (mp3, wav)，支持循环播放、音量控制、抽奖中段切换音轨。
- 舞台效果：开启/关闭烟花/五彩纸屑/字幕滚动、奖项出现动画（翻牌、滚动名单、光束等）。

### 3.4 抽奖引擎与公平性

- 支持两种抽奖模式：即时随机（实时抽取）与预先抽取（根据随机种子生成结果）。
- 权重抽样（带/不带替换）实现：支持按奖项权重或按候选人权重（例如VIP权重）。
- 使用安全随机数生成（服务端的加密 RNG）确保不可预测性。
- 提供抽奖日志（时间戳、随机种子、参与池快照、获奖人名单）以供审计和回放。

### 3.5 结果导出与回放

- 导出格式：Excel (.xlsx)，CSV，JSON。
- 每次抽奖的导出包含：活动ID、奖项、获奖人员、抽奖时间、随机种子、快照文件（可选）。
- 回放功能：根据日志在舞台上以相同动画顺序回放某次抽奖过程。

### 3.6 权限与多角色

- 角色：管理员、主持人（展现/控制抽奖）、审计员（只读）、访客（仅观看）。
- 登录方式：本地账号/密码、LDAP/AD、OAuth（可选）。

### 3.7 其他：稳定性与辅助功能

- 全屏展示与分辨率自适应（支持 16:9 / 4:3 / 自定义），深色/亮色皮肤。
- 离线模式（本地静态部署，上传资源缓存）。
- 多语言（中文/英文）与可配置文案。

------

## 4. 非功能性需求

- 性能：在 10,000 名单规模下，抽奖操作响应 < 1s（抽样计算），大屏展示动画流畅 60 FPS（取决于设备）。
- 可用性：单点故障情况下，支持重启后根据日志恢复状态。
- 安全：上传文件检查（类型、大小）、防止注入、对敏感导出支持权限控制。
- 可维护性：模块化代码、单元测试覆盖率目标 70%+、关键逻辑增加端到端自动化测试。
- 可扩展性：后端提供 RESTful API，前端组件化（可嵌入到其它页面）。

## 5. 技术选型建议

> 下列为推荐组合，可按团队栈做替换。

### 后端

- Node.js + Express / NestJS 或 ASP.NET Core（C#）
- 数据库：PostgreSQL（生产） + SQLite（轻量或本地部署）
- 文件/媒体存储：磁盘存储或对象存储（S3 兼容）
- 随机数：使用 Node.js 的 `crypto.randomInt` / 后端的加密 RNG

### 前端

- 框架：React 或 Vue3
- 动画：使用 Canvas / WebGL（Three.js）或 Lottie（bodymovin）实现高质演出动画
- Excel 解析：SheetJS (xlsx)
- 打包与部署：Vite 或 Webpack

### 其他

- 容器化：Docker
- CI/CD：GitHub Actions / GitLab CI
- 日志与监控：ELK / Prometheus + Grafana（可选）

## 7. API 设计（关键端点示例）

- `POST /api/events` — 创建活动
- `GET /api/events/:id` — 获取活动详情
- `POST /api/events/:id/import` — 导入名单（multipart upload）
- `POST /api/events/:id/prizes` — 新增奖项
- `POST /api/events/:id/draw` — 发起抽奖（返回结果并记录日志）
- `GET /api/events/:id/results` — 获取抽奖结果
- `POST /api/media` — 上传媒体
- `GET /api/export/:resultId` — 导出某次抽奖结果

## 8. 抽奖算法与实现细节

### 8.1 基本原则

- 使用服务端加密 RNG 生成随机数并记录随机种子（seed）。
- 对于带权重抽样，使用 **加权随机无替换抽样（Weighted Reservoir Sampling）** 或等价算法：把每个候选人分配 `key = -ln(U)/weight`（U∈(0,1]），取最小的 k 个作为获奖者。
- 对于简单按名额分配可采用扩展的抽样法（如随机打乱后按权重分配名额）。

### 8.2 可复现性与审计

- 抽奖时记录完整快照：候选池（或其哈希）、随机种子、时间戳、算法版本。
- 提供“公示页”：展示抽奖快照摘要与种子，外部方可用工具验证。

## 9. UI/UX 要求（高端视觉）

- 视觉基调：极简但有质感——深色主题 + 金属质感高光，使用大字号与动效突出“荣誉感”。
- 舞台页必须支持：奖项卡片、大号获奖姓名、部门小字、奖品图片/动画、背景光线与粒子效果。
- 音画同步：抽奖关键时刻（例如出现中奖者）伴随音效变化与视觉强调。
- 配置页使用表格/向导式导入体验，支持即时预览与回滚。

## 10. 开发里程碑与任务拆分（示例 8 周）

说明：以下为参考计划，按 1 周为一个短冲刺；可根据团队规模（2-4 人）调整。

- 集成动画（Lottie/Canvas/Three.js）

- 角色权限管理

- 单元测试与端到端测试

## 11. 测试计划

- 单元测试：抽奖算法、导入解析、导出逻辑。
- 集成测试：前后端交互、文件上传、权限控制。
- E2E 测试：完整导入→抽奖→导出场景自动化。
- 视觉回归测试（重要）：确保舞台动画在常用分辨率下无错位。
- 可用性测试：至少一次内部演练（10 人）验证流程与动画节奏。

------

## 12. 部署与运维

- 打包成 Docker 镜像，使用 Docker Compose 提供单机部署方案。
- 推荐云部署：Kubernetes + 对象存储（S3），并启用 HTTPS（Let’s Encrypt）。
- 备份策略：数据库每日全量备份/按需保存抽奖日志快照。

------

## 13. 安全和合规

- 文件上传限型与大小限制，病毒扫描/白名单后缀检查。
- 导出文件权限控制与审计链路。
- 对外公示页不包含敏感信息（如手机号），或对敏感字段做脱敏。
- 若公司在 EU/UK，考虑 GDPR 的数据保留与删除流程。

------

## 14. 风险与缓解措施

- 风险：大屏设备性能不足导致动画卡顿。缓解：提供“低性能模式”，降帧或简化动画。
- 风险：导入格式不规范导致漏人。缓解：提供导入模板与字段映射并在导入前做完整预览。
- 风险：抽奖争议。缓解：记录随机种子、公开快照、提供回放与第三方验证方法。

------

## 15. 验收标准（示例）

- 功能验收：完成需求内所有核心功能并通过回归测试。
- 性能验收：10k 名单规模下抽样 ≤1s；舞台在推荐硬件上动画 60FPS（或在低性能模式下平稳）。
- 安全验收：上传与导出流程无明显漏洞；权限控制验证通过。
- 体验验收：UI 通过品牌/设计方的视觉评审（颜色、字体、动效节奏）。
